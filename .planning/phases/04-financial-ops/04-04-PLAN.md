**Step 1 — Fix TS2345 errors in apps/api/src/chain/transfer.ts:**

The two `walletClient.writeContract` calls on lines 25 and 33 are missing the `chain` property. viem 2.x requires `chain` to be explicitly passed in `writeContract` parameters even when the walletClient was created with a chain. The fix: import the chain selection logic from clients.ts and pass `chain` explicitly.

Add a new import at the top of transfer.ts to get the chain:
```typescript
import { worldchain, worldchainSepolia } from 'viem/chains';
```

Then add a chain constant after the existing imports:
```typescript
const chain = process.env.WORLD_CHAIN_TESTNET === 'true' ? worldchainSepolia : worldchain;
```

Then add `chain` to BOTH writeContract call objects. The first call (line 25) becomes:
```typescript
const routeTxHash = await walletClient.writeContract({
  chain,
  address: GENIE_ROUTER_ADDRESS,
  abi: GenieRouterAbi,
  functionName: 'route',
  args: [senderWallet, amount, PAY_HANDLER_ADDRESS],
});
```

The second call (line 33) becomes:
```typescript
const executeTxHash = await walletClient.writeContract({
  chain,
  address: PAY_HANDLER_ADDRESS,
  abi: PayHandlerAbi,
  functionName: 'execute',
  args: [recipientWallet, amount],
});
```
