# Phase 07 Plan 03: Onboarding Threshold Wiring

## Objective

Wire the onboarding budget/threshold step to persist the user's auto-approve limit
to the backend via PATCH /api/users/profile. Currently the onboarding page collects
a monthly spending limit (autoApproveUsd) but only saves it to localStorage.

The gap: user sets a threshold during onboarding, but the agent's send_usdc tool
reads `autoApproveUsd` from the DB — so transfers never use the threshold the
user configured.

## Context

- Backend: PATCH /api/users/profile needs to be created in a new usersRoute
- Frontend: apps/web/src/app/onboarding/page.tsx finish() just calls router.push('/home')
- Auth: session.user.id is wallet address — backend must accept wallet address and resolve UUID
- DB: users.autoApproveUsd is the column to update

## Tasks

### Task 1: Create PATCH /api/users/profile backend route
type="auto"

**What:** Create `apps/api/src/routes/users.ts` with a PATCH `/profile` endpoint that
accepts `{ userId, autoApproveUsd }` and updates the user's threshold in the DB.

**Files to create:**
- `apps/api/src/routes/users.ts` — new route with PATCH /profile

**Files to modify:**
- `apps/api/src/index.ts` — mount usersRoute under /api

**Implementation:**
```typescript
import { Hono } from 'hono';
import { z } from 'zod';
import { db, users, eq } from '@genie/db';
import { resolveUserId } from './chat';
import { invalidateContextCache } from './chat';

export const usersRoute = new Hono();

const patchProfileSchema = z.object({
  userId: z.string().min(1),
  autoApproveUsd: z.number().positive().max(10000),
});

usersRoute.patch('/users/profile', async (c) => {
  const body = await c.req.json();
  const parsed = patchProfileSchema.safeParse(body);
  if (!parsed.success) {
    return c.json({ error: 'INVALID_INPUT', message: parsed.error.message }, 400);
  }

  const { userId: rawUserId, autoApproveUsd } = parsed.data;

  const userId = await resolveUserId(rawUserId);
  if (!userId) {
    return c.json({ error: 'USER_NOT_FOUND', message: 'Could not resolve userId' }, 404);
  }

  await db.update(users)
    .set({ autoApproveUsd: autoApproveUsd.toFixed(2) })
    .where(eq(users.id, userId));

  invalidateContextCache(userId);

  return c.json({ success: true });
});
```

**Done criteria:**
- `apps/api/src/routes/users.ts` exists with PATCH /users/profile handler
- `apps/api/src/index.ts` mounts usersRoute under /api
- Route returns 200 with { success: true } for valid input
- Route returns 404 when userId cannot be resolved

### Task 2: Wire onboarding finish to PATCH /api/users/profile
type="auto"

**What:** Update `apps/web/src/app/onboarding/page.tsx` so that the `finish()`
function (called from StepBudget "Let's Go" button) calls PATCH /api/users/profile
with the selected budget before navigating to /home.

The goals selection (StepGoals) is purely cosmetic — the selected goals do not need
to be persisted (there's no goals column in the DB). Only `autoApproveUsd` matters.

**Files to modify:**
- `apps/web/src/app/onboarding/page.tsx` — update finish() to call the API

**Implementation:**
The finish function receives the budget from the parent Onboarding component's state.
Update the function signature to accept the budget and make the API call:

```typescript
const finish = async (budgetValue: string) => {
  const apiUrl = process.env.NEXT_PUBLIC_API_URL ?? '';
  const userId = session?.user?.id;
  if (userId && budgetValue && budgetValue !== '0') {
    try {
      await fetch(`${apiUrl}/api/users/profile`, {
        method: 'PATCH',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          userId,
          autoApproveUsd: Number(budgetValue),
        }),
      });
    } catch (err) {
      console.error('[onboarding] failed to save threshold:', err);
      // Non-blocking: proceed to home even if API call fails
    }
  }
  localStorage.setItem('genie_onboarding_done', '1');
  router.push('/home');
};
```

The component needs `useSession` from `next-auth/react` to get the wallet address.
Update `StepBudget` to call `finish(budget)` instead of `finish()`.

**Done criteria:**
- Onboarding page imports useSession and reads session.user.id
- finish() calls PATCH /api/users/profile with userId and autoApproveUsd
- If API call fails, onboarding still completes (non-blocking)
- StepBudget's onFinish prop passes the budget value to finish
