This plan extracts that logic, builds the route, and wires everything together.

    import { erc20Abi, parseUnits, pad } from 'viem';
    import { getWalletClient, relayerAccount, chain, GENIE_ROUTER_ADDRESS, USDC_ADDRESS } from './clients';
    import { GenieRouterAbi } from '../contracts/abis';

    const TOKEN_MESSENGER_WORLD_CHAIN = '0x1682bd6a475003921322496e952627702f7823f9';

    Export `async function bridgeUsdc(params: { senderWallet: \`0x${string}\`; amountUsd: number; destinationChain: string; recipientWallet: string; }): Promise<{ routeTxHash: string; approveTxHash: string; bridgeTxHash: string }>`.

    The function body does 3 writeContract calls, each with explicit `account: relayer` and `chain` (imported from clients.ts, per viem 2.45 requirement):
    1. GenieRouter.route(senderWallet, amountUnits, relayer.address) -- pull from user to relayer
    2. USDC.approve(TOKEN_MESSENGER_WORLD_CHAIN, amountUnits) -- approve TokenMessenger
    3. TokenMessenger.depositForBurn(amountUnits, CCTP_DOMAIN_IDS[destinationChain], pad(recipientWallet, {size:32}), USDC_ADDRESS)

    Validate `destinationChain in CCTP_DOMAIN_IDS` at the top; throw Error `Unknown destination chain: ${destinationChain}` if not found.

    **Step B: Create `apps/api/src/routes/send.ts`** (per D-01, D-02)

    Create a Hono route `export const sendRoute = new Hono()` with POST handler:

    ```typescript
    import { Hono } from 'hono';
    import { isAddress } from 'viem';
    import { db, transactions, users, eq, and } from '@genie/db';
    import { executeOnChainTransfer } from '../chain/transfer';
    import { bridgeUsdc, CCTP_DOMAIN_IDS } from '../chain/bridge';
    import { inferCategory } from '../tools/categorize';
    ```

    Chain name mapping (backend normalization per Pitfall 3 in RESEARCH):
    ```typescript
    const CHAIN_MAP: Record<string, string | null> = {
      'World Chain': null,
      'worldchain': null,
      'Base': 'base',
      'base': 'base',
      'Arbitrum': 'arbitrum',
      'arbitrum': 'arbitrum',
      'Ethereum': 'ethereum',
      'ethereum': 'ethereum',
      'Optimism': 'optimism',
      'optimism': 'optimism',
    };
    ```

    Handler logic:
    1. Parse body: `{ userId, recipient, amount, chain: chainName, description? }`
    2. Validate: userId required (400), recipient must pass `isAddress(recipient)` (400 "Invalid recipient address"), amount must be positive number (400), chainName must be key in CHAIN_MAP (400 "Unsupported chain")
    3. Load user from DB (same pattern as confirm.ts): `const [user] = await db.select().from(users).where(eq(users.id, userId)).limit(1)` -- 404 if not found
    4. Verification gate: `if (user.worldId === null) return c.json({ error: 'VERIFICATION_REQUIRED', message: 'World ID verification required to send' }, 403)`
    5. Get autoApproveUsd: `const autoApproveUsd = parseFloat(user.autoApproveUsd)`
    6. Determine route: `const destinationChain = CHAIN_MAP[chainName]`

    **If destinationChain === null (World Chain, per D-04):**
    - If `amount <= autoApproveUsd` (FOPS-04): call `executeOnChainTransfer(user.walletAddress, recipient, amount)`, insert confirmed tx into DB with `source: 'genie_send'`, `category: inferCategory(description)`, return `{ type: 'transfer_complete', txHash: executeTxHash, routeTxHash, amount, recipient }`
    - If `amount > autoApproveUsd` (FOPS-05): expire existing pending txs for user, insert pending tx with 15min expiry, return `{ type: 'confirmation_required', txId: pending.id, amount, recipient, expiresInMinutes: 15 }`

    **If destinationChain is a string (cross-chain, per D-05, D-08):**
    - Call `bridgeUsdc({ senderWallet: user.walletAddress, amountUsd: amount, destinationChain, recipientWallet: recipient })`
    - Insert confirmed tx with `source: 'genie_bridge'`, `category: 'transfers'`
    - Return `{ type: 'bridge_initiated', routeTxHash, bridgeTxHash, amount, recipient, destinationChain, estimatedMinutes: 15 }`

    Wrap everything in try/catch. On error return `{ error: 'SEND_FAILED', message: err.message }` with 500.
